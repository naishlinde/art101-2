Lab 10: JavaScript Events & Forms

Challenges

Our challenge was to create a basic website that featured an input form field and a button. In javascript, we are supposed to use the scrambling function from Lab 7 using the user input from the form field. This function would then be activated by the button by adding an event listener. This function would first separate every character into an array of strings using .split, then shuffle each string using .sort, finally, it would rejoin the array into a single string by using .join. The results of our user’s submission would then be printed below the input form field and button as a new element. The main challenges of this lab were altering the javascript to do all of the functions of Lab 7 using an input form field rather than a window prompt appearing when the user first opens the webpage.

Problems

The lack of description in the Canvas assignment made this assignment especially difficult. We had plenty of trouble sorting out the semantics and keeping uniformity between our HTML and JavaScript files. When paring the data form field with the JavaScript, we gave the form field the id="user-name" in the HTML file and referenced it with document.getElementById(“my-input”). By using a different reference, the js did not sufficiently link up to the HTML, thus the following console.log did not print.

Once we had our HTML and JavaScript files configured correctly, we found that the function would print the new name twice on the website. After some analysis we found that the issue was that we were assigning the functions output to a variable and then appending the variable to the output div which inserted the output to the document while triggering the code to run again causing it to reprint the output. By referencing the button element in the new variable, we were able to send the function output back to the button element which integrated the output.
